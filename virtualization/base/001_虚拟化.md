
### 虚拟化分类

CPU 如果想要可虚拟化，所有的敏感指令必须是特权指令；这样在执行敏感指令时，会触发陷入（trap），从而被 VMM 捕获处理

在早期的 x86 架构下，存在某些指令是敏感指令，但不是特权指令；所以这些 guest 的敏感指令执行时不会陷入，会直接在 CPU 上执行


* 全虚拟化（Full Virtualization）

扫描 guest 内核代码，通过二进制翻译（Binary Translation），将所有敏感指令都替换成跳转到 VMM 代码，这样这些指令可以在 VMM 中进行处理


* 半虚拟化（Para virtualization）

修改 guest 内核，修改敏感非特权的指令，替换成主动调用由 VMM 提供的 hypercall 调用接口


* 硬件辅助虚拟化（Hardware-assisted Virtualization）

后来 CPU 厂商支持虚拟化，例如 intel 使用了 VT 技术

这种 CPU 有 root 和 non-root 两种模式，root 模式可以执行敏感指令，non-root 不允许执行敏感指令

VMM 运行在 root 模式下，guest 运行在 non-root 模式下

当 guest 试图执行敏感指令，CPU 会陷入到 root 模式，交给 VMM 来处理，这个过程叫 vm exit

VMM 处理完后，会继续运行虚拟机，这个过程叫 vm entry


### 虚拟化技术

* Intel VT: Intel Virtualization Technology

x86 架构的虚拟化技术，称为 VT-x

Itanium 架构（Intel 公司另一种架构 CPU）的虚拟化技术，称为 VT-i


* AMD SVM: Secure Virtual Machine

也叫 AMD-V


### 虚拟化层级

客户机 - 虚拟硬件抽象层 - 虚拟化层 - 硬件抽象层 - 硬件

虚拟化层被称为虚拟机监控器（virtual machine monitor，即 VMM）
