# 版本回退


### 查看历史提交日志

```bash
git log

# 每次提交的内容只输出一行结果
git log --oneline

git log --oneline --graph
```


### 丢弃

* 丢弃未跟踪的文件（没有 git add 的文件）

```bash
git clean -fd
```

* 丢弃工作区的修改

```bash
# 当前目录下
git checkout -- .

# 具体目录
git checkout -- $path
```

* 丢弃工作区和缓存区的修改

```bash
 # 返回到最后一次 commit
git reset --hard HEAD^

# 返回前一百次 commit
git reset --hard HEAD~100

git reset --hard 版本号
```


### 回退

--hard：回退到某个版本，丢弃暂存区和工作区修改

--mixed：会退到某个版本，把缓存区的内容回退到工作区，工作区的修改保留

--soft：回退到某个版本，缓存区的内容保留，工作区的内容保留

* 已经添加到暂存区的文件撤回到工作区

如果 git reset 不加参数，默认是 --mixed

```bash
# 某个文件
git reset 文件名

# 所有文件
git reset
```

* 已经提交的撤回到暂存区

```bash
git reset --soft HEAD^
```


### 撤销回退

* 查看操作的历史

```bash
# 撤销回退时候，如果找不到版本号，可以使用下面的命令
git reflog
```


### 待验证

* 删除一个文件后，如果想在版本库里也删除

```bash
# 从缓存区删除文件
git rm 文件名

# 从工作区删除
git rm --cache 文件名

# 递归删除
git rm -f *

git commit
```

* 如果删错了，想恢复

```bash
git checkout -- 文件名
```
