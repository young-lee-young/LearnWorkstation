# SVN基础操作


### 服务端命令

```sh
# 开启svn服务,默认3690端口
svnserve -d -r 版本库路径
svnserve -d -r 版本库路径 --listen-port 3691
```


### 客户端操作

```sh
# 检出版本库,存在.svn文件
svn checkout -r 版本号 svn://地址 --username 用户名 --password 密码
命令参数: -r 版本号

# 导出版本库,无版本控制
svn export -r 版本号 版本库地址

# 添加到版本控制
svn add *       # 添加所有文件
svn add * --force
svn add 文件或文件夹(文件夹内的文件会递归加入)
svn add 文件夹 --non-recursive(只加入文件夹,不加入里面的文件)

# 提交到版本库
svn commit -m '提交备注' 文件名

# 更新版本库
svn update
svn update -r 版本号 文件名
svn update *

# 删除文件
svn delete/del/remove/rm 文件名 -m '删除备注'

# 版本比较
svn diff 文件名                # 工作版本和最新版本
svn diff -r 版本号 文件名       # 工作版本和指定版本
svn diff -r 版本号:版本号       # 两个版本比较

# 创建的同时添加
svn mkdir 文件名

# 不检出工作副本直接查看指定文件
svn cat svn://地址/文件名

# 工作副本还原
svn revert 文件名
svn revert *
svn revert --recursive *       # 子目录递归还原

# 冲突与处理
svn resolve 文件名              # 处理
svn resolved 文件名             # 处理冲突完成

# 锁定与解锁
svn lock 文件名
svn unlock 文件名               # 如果有提交的动作,提交完成后自动解锁 
svn commit -m '不自动解锁' --no-unlock 文件名
```
