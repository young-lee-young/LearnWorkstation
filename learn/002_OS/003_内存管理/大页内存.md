# 大页内存


### What：什么是大页内存

普通内存一个页大小为 4KB，大页内存每个页为 2MB 或者 1GB 


### Why：为什么要用大页内存

* 优点

1. 减少页表项数量

1GB 内存，如果每个页是 4KB，需要 262144 个页表项，如果每个页是 2MB，只需要 512 个页表项

页表项减少，页表占用的内存减少

2. 降低 TLB miss 开销

TLB 的容量有限（通常 60 - 1024 个），小页时，页表项非常多，TLB miss 的概率高，大页情况下，页表项少，TLB miss 的概率低

miss 少了，CPU 不需要频繁去访问页表，内存访问的性能变高


* 缺点

1. 内存分配问题

内存页要求物理内存连续，如果是 2MB 的大页，物理内存可能不够，可能会导致内存分配失败

2. 浪费问题

每个页大小 2MB，可能只使用了很小一部分，造成浪费


### How：怎么使用大页内存

* 系统自动管理（透明大页）

```bash
cat /sys/kernel/mm/transparent_hugepage/enabled

# 输出：always [madvise] never
# always：内核尽可能使用大页
# madvise：程序通过 madvise 调用时才使用
# never：禁用
```

```c
// madvise 调用
madvise(addr, length, MADV_HUGEPAGE);
```


* 显式大页

显式使用大页分成 3 个步骤

1. 配置系统的 hugepage 数量

```bash
# 分配 128 个 2MB 的大页
echo 128 | sudo tee /proc/sys/vm/nr_hugepages

# 写入 /etc/sysctl.conf 永久生效
echo "vm.nr_hugepages=128" | sudo tee -a /etc/sysctl.conf
```

```bash
# 配置超额的部分
# 允许超过分配大页的数量，比如分配了 128 个大页，那么还可以尝试分配 64 个大页，但是不保证成功
echo 64 | sudo tee /proc/sys/vm/nr_overcommit_hugepages
echo "vm.nr_overcommit_hugepages=64" | sudo tee -a /etc/sysctl.conf
```

```bash
# 查看大页信息
cat /proc/meminfo

# HugePages_Total：总大页数量
# HugePages_Free：剩余大页数量
# HugePages_Rsvd：已经预留给程序，但是还没有真正使用的大页数量
# HugePages_Surp：超额大页数量，由 nr_overcommit_hugepages 控制上限

# Hugepagesize：大页的大小
```

⚠️：已经被分配为大页内存的这部分内存不可以再被普通的程序（使用 4KB 页的程序）使用

2. 挂载 hugetlbfs 文件系统

```bash
sudo mount -t hugetlbfs none /mnt/hugepages

# 如果需要 1GB 大页，需要加参数
sudo mount -t hugetlbfs -o pagesize=1G none /mnt/hugepages
```
