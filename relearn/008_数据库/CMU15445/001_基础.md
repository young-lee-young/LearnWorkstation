# 数据库基础


### Data Model（数据模型）

1. Relational（关系型）
2. Key/Value
3. Graph（图）
4. Document/JSON/XML/Object（文档，如 NoSQL）
5. Array（向量）


### Relational Model（关系模型）

* 什么是关系模型

The relational model defines a database abstraction based on relations to avoid maintenance overhead

关系模型定义了基于关系的数据库抽象，减少维护的开销

* Key Tenets（关键原则）

1. 将数据库存储为简单的数据结构
2. 物理存储由 DBMS 来实现
3. 通过高级语言来访问数据，DBMS 来找到最佳的执行策略


### 关系模型的概念

* 关系：可以理解为表结构

* tuple（元组）：可以理解为一行数据，元组的每个值为一个列


### Data InDependence（数据独立性）

Database Storage -> Physical Schema -> Logical Schema -> External Schema

Database Storage：存储：磁盘、内存
Physical Schema（物理模式）：例如数据文件是如何组织的，定义页面和页面的布局
Logical Schema（逻辑模式）：有哪些表、哪些列，哪些约束，一般是通过 SQL 来操作
External Schema（外部没收）：一般指视图（View，在表的基础上定义虚拟表）


### DML（Data Manipulation Languages，数据操作语言）

* 作用

DBMS 暴露给应用的 API，用于存储和查询数据

* 类型

Procedural：过程式

Non-Procedural（Declarative）：非过程式，声明式
